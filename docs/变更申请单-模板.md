# 变更申请单（Change Request）模板

> 关联文档：`docs/03-configuration-management-plan.md` §6
>
> 使用方式：复制本文件到 `docs/变更申请单-实例/CR-YYYYMMDD-NNN-<简述>.md` 后填写。

---

## 一、变更标识

| 字段 | 值 |
|---|---|
| **CR ID** | CR-YYYYMMDD-NNN |
| **申请日期** | YYYY-MM-DD |
| **申请人** | ___ |
| **所属角色** | 开发员 / 测试员 / 配置管理员 / 用户 |
| **来源分支 / 基线** | `phase-XX-xxx` / `v1.0.0-mX` |
| **变更类别** | ☐ A 类（轻微） ☐ B 类（一般） ☐ C 类（重大） |

## 二、变更内容描述

### 2.1 现状

> 描述当前系统的行为、涉及的配置项 ID（CI-XX-NNN）、相关基线。

### 2.2 期望

> 描述变更后期望的行为。

### 2.3 变更理由

> 为什么需要这个变更？业务诉求 / 缺陷修复 / 安全问题 / 技术债？

## 三、影响分析

| 维度 | 分析 |
|---|---|
| 涉及配置项 | CI-XX-NNN（列出全部受影响的 CI） |
| 涉及基线 | 是否修改已建立的基线？ |
| 接口影响 | 是否改变对外 API / DB schema / 配置文件结构？ |
| 风险等级 | 高 / 中 / 低，并说明依据 |
| 回滚方案 | 若变更失败如何回滚（切 tag / 还原 commit / 数据库回滚） |
| 测试影响 | 需要新增或修改哪些测试？ |
| 文档影响 | 需要同步更新哪些文档？ |
| 预估工作量 | 人时 |

## 四、CCB 审批

| 字段 | 值 |
|---|---|
| **CCB 会议日期** | YYYY-MM-DD |
| **参会成员** | CM 经理 / 技术负责人 / 测试负责人 |
| **决议** | ☐ 批准 ☐ 驳回 ☐ 待补充材料后再审 |
| **附加条件** | ___ |
| **审批人签字** | ___ |

## 五、实施记录

| 字段 | 值 |
|---|---|
| **实施分支** | `feature/CR-YYYYMMDD-NNN-xxx` 或对应 phase 分支 |
| **实施人** | ___ |
| **完成日期** | YYYY-MM-DD |
| **关联 commit** | `<SHA>` `<SHA>` |
| **测试结果** | CI 全绿？回归测试？覆盖率达标？ |
| **配置审计结果** | PCA 通过？FCA 通过？ |
