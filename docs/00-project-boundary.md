# 项目边界

## 1. 当前定位

PetCare O2O V1 是单门店、模块化单体、H5 优先的宠物门店业务应用。

- 用户主入口：响应式 H5 Web。
- 管理入口：PC Web。
- 微信登录和小程序：后续兼容目标。
- AI：当前禁用和延后。

## 2. 当前必须保留的业务

- 用户资料、宠物档案和地址。
- 服务展示、预约、排班、距离和并发控制。
- 商品、购物车、订单金额、库存和状态流转。
- 社区发帖、浏览、点赞、评论、收藏和基础审核。
- 基础营销活动的展示、管理及商品/服务关联。
- 管理端权限和业务处理。

## 3. 当前明确不做

- 微服务、多门店和独立员工端。
- 在线支付、退款、优惠券、会员积分和配送系统。
- 复杂社交关系、私信、推荐算法和直播。
- AI 助手、AI 内容生成、AI 营销和 AI 直连数据库。
- 宠物疾病诊断、药物处方或治疗承诺。

## 4. 不得改变的核心规则

- 价格、库存、订单金额、预约容量和状态流转由服务端决定。
- 商品下单和多表状态变更必须使用事务。
- 预约必须校验距离、时段容量、重复提交和并发冲突。
- 用户只能访问和修改自己的私有数据。
- 公开接口必须使用专门的公开响应模型，不能直接暴露内部实体。
- 管理权限必须由后端强制执行。
- 所有外部输入必须校验，所有秘密必须来自安全配置。

## 5. 范围变更

新增功能只有在直接支持当前 H5 核心流程或修复真实风险时才能进入 V1。任何会引入新平台、新支付链路、新业务角色或复杂运营规则的需求，先记录到 `docs/08-pending-decisions.md`，不得顺手实现。
