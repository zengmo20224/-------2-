## 门禁 8：AI 安全

- [ ] 已按 `docs/17-phase-8-ai-provider-functions-plan.md` 执行。
- [ ] AI Provider 位于统一适配器之后。
- [ ] Provider 可以被 Mock。
- [ ] 默认 Provider 关闭，关闭时不会伪造 AI 成功响应。
- [ ] Provider 关闭或不可用时返回安全错误。
- [ ] Provider 包不依赖 Mapper、DataSource、JdbcTemplate 或 MyBatis。
- [ ] Provider 请求不包含 SQL、数据库连接或工具调用定义。
- [ ] 用户不能通过请求体指定 `userId`、`adminId`、模型、系统 Prompt 或 Provider。
- [ ] 用户只能读取和继续自己的 AI 会话。
- [ ] 会话、用户消息、最终安全助手消息按规则持久化。
- [ ] 每次 Provider 调用尝试写入脱敏后的成功或失败用量日志。
- [ ] AI 客服由后端上下文接地。
- [ ] 没有可信上下文时不会编造价格、库存、营业时间、服务半径或取消规则。
- [ ] AI 客服上下文只来自批准的数据源。
- [ ] Prompt 注入不能获取密钥、内部配置或启用工具调用。
- [ ] 高风险宠物症状由确定性规则直接建议就医。
- [ ] 命中高风险宠物症状时不调用 Provider。
- [ ] 诊断、处方、自行用药和治疗承诺输出被阻止。
- [ ] AI 发帖辅助不编造用户未提供事实。
- [ ] AI 发帖辅助不自动发布帖子或改变审核状态。
- [ ] 管理端分析使用后端聚合数据。
- [ ] 管理端分析不能执行 AI 生成 SQL。
- [ ] AI 输出不能直接修改核心业务状态。
- [ ] 管理端分析和用量接口使用细粒度权限码。
- [ ] 管理端生成分析报告成功和失败都写操作日志。
- [ ] 高风险宠物症状会触发建议就医。
- [ ] AI 不能直接查询数据库。
- [ ] Provider 原始错误、完整 Prompt、API Key 和 Authorization Header 不会出现在响应或日志中。
- [ ] D-008 未决项未决定前，没有写死模型、超时、重试、最大 Token、限流或流式策略。
- [ ] D-008 未决项未决定前，没有声称真实 DeepSeek 已接入。

- [ ] `mvn clean package` 通过。
