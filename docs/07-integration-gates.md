# 集成门禁

日期：2026-06-08

任何阶段在通过对应门禁前，都不能算真正完成。

## 门禁 0：Agent 开工检查

- [ ] 已阅读根目录 `AGENTS.md`。
- [ ] 已阅读原始需求、项目边界和架构设计。
- [ ] 已阅读 `docs/08-pending-decisions.md`。
- [ ] 已运行 `git status --short --branch`。
- [ ] 已汇报当前阶段、修改范围、不修改范围和验证计划。
- [ ] 当前任务不依赖未决事项，或已获得用户明确决定。

## 门禁 1：Schema

- [ ] `schema.sql` 存在。
- [ ] 所有必需表都存在。
- [ ] 细粒度 RBAC 的 `admin_role`、`admin_permission`、`admin_role_permission` 存在。
- [ ] SQL 可以导入干净的 MySQL 8 数据库。
- [ ] 表注释存在。
- [ ] 重要字段注释存在。
- [ ] 唯一索引存在。
- [ ] 高频查询索引存在。
- [ ] 状态、创建时间、更新时间、deleted 字段默认值存在。
- [ ] 主键不使用 `AUTO_INCREMENT`，符合 MyBatis-Plus 雪花 ID 决策。
- [ ] 没有引入 Flyway、Liquibase 或 Redis。
- [ ] `staff_booking_lock` 表存在，且 `(staff_id, booking_date)` 唯一索引存在。
- [ ] Git 提交存在。

## 门禁 2：后端骨架

- [ ] 已按 `docs/11-phase-2-backend-skeleton-brief.md` 执行。
- [ ] Spring Boot 应用可以启动。
- [ ] 健康检查接口可用。
- [ ] 数据库连接通过环境配置。

## 门禁 3：核心业务逻辑

- [ ] 已按 `docs/12-phase-3-entities-mappers-plan.md` 执行。

- [ ] 单元测试覆盖纯计算逻辑。

- [ ] Git 提交存在。

## 门禁 4：安全

- [ ] 后台 API 必须认证。
- [ ] 角色授权已生效。
- [ ] 使用权限码进行细粒度授权。
- [ ] 用户资源归属已校验。
- [ ] 密码已哈希。
- [ ] Provider 密钥来自环境变量。
- [ ] 错误不会泄露堆栈或 SQL 细节。
- [ ] Git 提交存在。

## 门禁 5：AI 安全

- [ ] AI Provider 位于统一适配器之后。
- [ ] Provider 可以被 Mock。
- [ ] AI 客服由后端上下文接地。

- [ ] 高风险宠物症状会触发建议就医。
- [ ] AI 不能直接查询数据库。
- [ ] Git 提交存在。
