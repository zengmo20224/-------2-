## 二、变更内容描述

### 2.1 现状

- 项目已完成 M1–M6 里程碑的代码交付，但无配置管理计划（CMP）。
- 无 git tag 基线，里程碑仅以 commit message 体现，无法 `git checkout` 还原历史基线。
- 无 CI/CD 流水线（Jenkinsfile、GitHub Actions 均不存在）。
- 无 Docker 化，无部署脚本。
- 无配置项登记表、无变更控制记录。
- **直接导致课程评分中"配置管理计划 30 分 + 实施证据 15 分 + 自动化 30 分"共 75 分无法获得。**

### 2.2 期望

按 IEEE 828 建立完整配置管理资产，满足课程评分标准：

1. **配置管理计划**：`docs/03-configuration-management-plan.md`（主文档）+ 配置项登记表 + 基线清单 + 变更申请单模板。
2. **基线管理**：为 M1–M6 打 git tag（`v1.0.0-m1` ... `v1.0.0-m6`）+ 产品基线 `v1.0.0-rc1`。
3. **CI/CD 自动化**：Jenkins（主力）+ GitHub Actions（补充）实现自动编译、打包、测试、部署。
4. **容器化部署**：Dockerfile（后端 + 两个前端）+ docker-compose.yml 单机编排。
5. **配套文档**：构建指导书 `docs/06-build-guide.md` + 部署指南 `docs/07-deployment-guide.md` + CHANGELOG.md。

### 2.3 变更理由

- 课程期末大作业核心评分项（占 75 分）。
- 落地后可显著提升项目工程成熟度，为后续迭代奠定基础。
