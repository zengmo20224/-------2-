## 二、变更内容描述

### 2.1 现状

M0 基线建立时，H5 公开登录方式未明确，存在以下风险：
- 未明确认证方式（手机号 / 邮箱 / 微信），开发无法推进。
- 找回密码机制缺失，账号丢失后无补救路径。
- 影响 M3「用户资料与真实预约」里程碑的退出条件。

涉及配置项：CI-SC-001（后端源码）、CI-SC-005（H5 源码）、CI-DOC-013（需求基线）。

### 2.2 期望

明确 V1 公开登录采用 **手机号 + 密码**，找回密码采用 **安全问题** 机制：
- 用户注册时设置安全问题与答案。
- 忘记密码时回答安全问题验证身份后重置密码。

### 2.3 变更理由

- 微信登录延后（决策 D-003），必须有可立即落地的登录方案。
- 邮箱在国内 C 端场景使用率低。
- 安全问题是最低实现成本的找回机制，符合 V1「最小可用」原则。
- 直接阻塞 M3 预约流程，必须立即决策。
