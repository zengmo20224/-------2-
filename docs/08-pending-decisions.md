# 前期待决策事项

日期：2026-06-08

这些事项目前仍不明确。任何 AI Agent 都不能擅自选择，必须由用户确认后才能进入相关实现。

## 决策状态说明

- `未决`：禁止实现相关内容，必须先询问用户。
- `已决定`：可以按记录的决定实现。
- `暂缓`：当前版本不实现。

## D-001：后端认证框架

状态：未决

可选方案：

- Sa-Token
- Spring Security + JWT

影响：

- 登录流程
- Token 管理
- 权限注解
- 测试方式
- 后续微信登录接入

推荐：如果项目更偏简历展示和标准企业技术栈，选择 Spring Security + JWT；如果优先快速实现，选择 Sa-Token。

## D-002：主键生成策略

状态：未决

可选方案：

- MyBatis-Plus 雪花 ID
- 数据库 `AUTO_INCREMENT`

影响：

- `schema.sql`
- 实体类配置
- 分布式扩展能力
- 测试数据构造

推荐：单体 V1 优先使用 MyBatis-Plus 雪花 ID，并在代码层生成主键；但必须在生成 `schema.sql` 前确认。

## D-003：数据库版本迁移工具

状态：未决

可选方案：

- 仅维护 `schema.sql`
- Flyway
- Liquibase

影响：

- 阶段 2 后的数据库升级流程
- CI/CD
- 回滚和环境一致性

推荐：阶段 1 使用 `schema.sql`，阶段 2 引入 Flyway 管理后续增量迁移。

## D-004：预约并发防冲突方案

状态：未决

可选方案：

- 数据库事务内再次检查 + 合理索引
- 悲观锁
- 乐观锁
- Redis 分布式锁

影响：

- 同一员工同一时间段重复预约风险
- 数据库事务设计
- 是否需要提前引入 Redis

推荐：V1 使用数据库事务内二次冲突检查；不因该功能提前引入 Redis。并发模型和隔离级别必须在阶段 5 前确认。

## D-005：管理员权限粒度

状态：未决

当前需求角色：

- `SUPER_ADMIN`
- `MANAGER`
- `STAFF`

仍需明确：

- `STAFF` 可以访问哪些后台模块。
- `MANAGER` 是否可以管理员工和管理员账号。
- 谁可以执行删除、下架、确认收款和内容审核。

影响：

- 权限模型
- API 授权注解
- 操作日志
- 测试用例

## D-006：微信登录开发模式

状态：未决

仍需明确：

- V1 是否接入真实微信开发者 AppID。
- 本地开发是否提供模拟登录。
- 模拟登录是否只允许在 `dev` profile 中启用。

推荐：先实现只在开发环境启用的模拟登录，预留真实微信登录适配接口。

## D-007：文件上传限制

状态：未决

仍需明确：

- 单文件大小限制。
- 图片格式白名单。
- 每个帖子最多上传多少张图片。
- 本地上传目录。
- 是否需要缩略图和图片压缩。

推荐初始值：

- 单文件最大 5 MB。
- 仅允许 JPG、PNG、WebP。
- 每个帖子最多 9 张图片。
- V1 使用本地上传目录。

## D-008：AI Provider 接口协议

状态：未决

仍需明确：

- Provider 是否兼容 OpenAI API。
- Base URL、模型名称和密钥环境变量名。
- 超时、重试次数、最大 Token。

## D-009：V1 是否启用 Redis

状态：暂缓

当前决定：

- V1 核心业务不依赖 Redis。
- 预约冲突防护优先使用数据库事务。
- 只有明确出现缓存、共享限流或分布式锁需求时才重新评估。

## D-010：前端实施顺序

状态：未决

可选方案：

- 后端核心 API 完成后，先做管理后台。
- 后端核心 API 完成后，先做小程序端。
- 按业务模块同时推进两端。

推荐：先完成管理后台核心运营流程，再完成小程序用户流程，因为预约确认、排班和内容审核依赖后台能力。

## 决策记录规则

用户确认某项决策后，负责执行的 Agent 必须：

1. 将状态改为“已决定”。
2. 写明选择结果、日期和理由。
3. 更新受影响的架构、任务和实施计划文档。
4. 提交独立 Git 文档提交。
5. 再开始相关代码实现。
