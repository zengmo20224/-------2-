# PetCare O2O 当前产品需求基线

更新日期：2026-06-13

本文件取代历史需求长文，作为当前 V1 产品需求基线。历史原始需求仍可从 Git 历史查阅。

## 1. 产品目标

为单体宠物门店提供一个真实可用的线上业务入口，让用户可以通过响应式 H5 完成浏览、预约、购物、订单查询和社区互动，让门店通过管理端完成日常运营。

当前成功标准不是功能数量，而是核心流程可真实运行、数据一致、移动端体验可用。

## 2. 用户与终端

- 普通用户：通过手机或桌面浏览器访问 H5。
- 门店管理员：通过 PC 管理端处理业务。
- 微信用户：后续增加微信登录和小程序适配，不阻塞当前 H5。

## 3. V1 必须交付

### 3.1 公开浏览

- 浏览服务、商品、社区帖子和营销活动。
- 支持列表、详情、分页、搜索或基础筛选。
- 公开接口不得泄露用户手机号、内部审核信息或管理字段。

### 3.2 用户资料

- 登录后维护个人资料、宠物档案和服务地址。
- H5 正式公开登录方式尚未确定；开发和验收阶段可使用现有测试身份入口。
- 未确定正式登录方式前，不得声称已具备公开生产登录能力。

### 3.3 预约

- 选择服务、日期、时段、宠物和地址并创建预约。
- 服务端校验服务状态、地址距离、容量、重复提交和状态流转。
- 预约并发控制必须真实生效。

### 3.4 商品与订单

- 商品列表和详情使用真实后端数据。
- 用户可维护购物车、提交订单、查看订单和执行允许的订单操作。
- 订单金额、商品快照、库存和状态由服务端校验并事务处理。
- 不接在线支付；支付状态只按已批准的线下业务流程处理。

### 3.5 社区

- 支持发帖、浏览、点赞、评论和收藏。
- V1 不实现复杂关注关系、私信、推荐算法或 AI 内容生成。
- 必须保留基础内容审核和公开数据隐私保护。

### 3.6 营销活动

- 用户可查看活动列表和详情。
- 管理员可维护基础活动，并关联商品或服务。
- V1 不实现优惠券、积分、复杂折扣引擎、自动投放或 AI 营销。

### 3.7 管理端

- 管理服务、预约、商品、订单、社区内容和营销活动。
- 权限和状态流转必须由后端校验，前端隐藏按钮不能代替授权。

## 4. 延后范围

- 微信登录、微信小程序专项适配。
- AI 助手、AI 文案、AI 审核和 AI 营销。
- 在线支付、退款、优惠券、会员积分。
- 多门店、独立员工端、配送系统和复杂报表。
- 宠物疾病诊断、药物处方或治疗承诺。

## 5. 核心验收流程

1. 未登录用户在 H5 浏览服务、商品、帖子和活动。
2. 测试身份完成个人资料、宠物和地址维护。
3. 用户完成一次真实预约并在管理端处理状态。
4. 用户将真实商品加入购物车并提交订单，管理端处理订单状态。
5. 用户发布帖子，并完成点赞、评论和收藏。
6. 管理员发布活动，用户在 H5 查看活动及其关联内容。

每个流程都必须有可重复的验证证据，不能只展示静态页面。
